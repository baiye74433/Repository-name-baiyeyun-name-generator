# SITE_QUALITY_CHECKLIST

项目：`baiyeyun.xyz`

## 用途
每次大批量新增页面后，先跑这份清单，再继续扩站或申请广告。

## 1. 数字一致性
- 首页工具数是否等于真实工具页数量
- `AI_HANDOFF.md` 数字是否同步
- `RETURN_LATER.md` 数字是否同步
- `PROJECT_DEV_LOG.md` 数字是否同步
- `DEPLOY_LOG.md` 数字是否同步

## 2. 入口完整性
- 新工具是否进了 `tools/index.html`
- 新博客是否进了 `blog/index.html`
- `sitemap.xml` 是否包含新 URL
- 首页是否仍能顺利进入主要分类

## 3. 页面质量
- 标题没有写错类别
- 描述没有和别的页面完全重复
- `Related tools` 指向对口页面
- `Keep browsing` / 索引推荐逻辑合理
- 没有明显占位文案
- 没有空白 FAQ / 空白列表

## 4. 基础页面
- `about.html`
- `contact.html`
- `privacy.html`
- `terms.html`
- `404.html`
这些页面都要能正常打开。

## 5. 申请广告前重点
- 不是只堆数量，最近 20 个页面要抽查质量
- 内链不能乱指
- 不要出现“首页写 60，实际 78”这种错
- 博客索引和工具索引都要能覆盖最新内容
- 等 Cloudflare 部署完成再去申请或提交审核

## 6. 推荐动作顺序
1. 先核对真实数量
2. 再改统计文档
3. 再抽查最近新增页面
4. 最后 push 并等待部署

## 当前提醒
- 这个项目后面最容易出的问题，不是技术坏掉，而是“页面越堆越多，统计和索引没跟上”。
- 所以后面每轮先看数字，再加页。
